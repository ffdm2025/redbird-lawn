Overwrite `astro.config.mjs` with the complete verified config below. This adds `@tailwindcss/vite` in `vite.plugins` and the `fonts:` top-level array.

CRITICAL ANTI-PATTERNS — do NOT use:
- `experimental: { fonts: [...] }` — outdated, causes errors in Astro 6.1.3
- `@astrojs/tailwind` in integrations — deprecated for Tailwind 4
- Any adapter (`@astrojs/cloudflare`) — SSR only, breaks static output

```typescript
import { defineConfig, fontProviders } from 'astro/config';
import react from '@astrojs/react';
import sitemap from '@astrojs/sitemap';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  site: 'https://redbirdlawnservice.com',
  output: 'static',
  integrations: [
    react(),
    sitemap(),
  ],
  vite: {
    plugins: [tailwindcss()],
  },
  fonts: [
    {
      provider: fontProviders.fontsource(),
      name: 'Space Grotesk',
      cssVariable: '--font-heading',
      weights: [400, 600, 700],
      styles: ['normal'],
      subsets: ['latin'],
    },
    {
      provider: fontProviders.fontsource(),
      name: 'Inter',
      cssVariable: '--font-body',
      weights: [400, 500],
      styles: ['normal'],
      subsets: ['latin'],
    },
  ],
});
```

**Step 2: Overwrite `src/styles/global.css`**

Replace the Plan 01 placeholder with the full Tailwind + font variable setup. The `@import "tailwindcss"` MUST be the first non-comment line:

```css
@import "tailwindcss";

/* Font family CSS custom properties — values injected by Astro Fonts API at build time */
:root {
  font-family: var(--font-body, system-ui, sans-serif);
}

h1, h2, h3, h4, h5, h6 {
  font-family: var(--font-heading, system-ui, sans-serif);
}

/* Dark mode variant — Tailwind 4 syntax */
@variant dark (&:is(.dark *));
```
